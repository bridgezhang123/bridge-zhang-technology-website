# Collaboration / 协作说明

如果您愿意参与本站内容的完善，非常欢迎。无论是补充案例、修正表述、指出疏漏，还是提出新的实践经验，都会对这个站点有所帮助。

!!! tip "先说明一点"
    您不需要一次性做很多，也不需要担心表达是否“足够正式”。哪怕只是指出一个错误、补充一条经验，都是有价值的。

## Ways to Contribute / 参与方式

您可以通过以下方式参与：

1. 在 GitHub 提交 `Issue`，反馈问题、建议或勘误。
2. 在 GitHub 提交 `Pull Request`，直接修改页面内容。
3. 通过邮件联系，提交建议、材料或投稿意向。

## A Simple GitHub Workflow / GitHub 参与的简单流程

如果您希望通过 GitHub 参与，可参考下面这条简化流程：

1. 打开仓库并 `Fork` 到自己的 GitHub 账户。
2. 在自己的副本中创建分支，修改相应的 Markdown 文件。
3. 提交修改，并发起 `Pull Request`。
4. 我会阅读、合并，或在必要时继续交流细节。

如果您对 GitHub 操作还不熟悉，也完全没有关系。直接通过邮件说明想法，同样欢迎。

## What Kind of Help Is Especially Welcome / 特别欢迎的内容

以下内容尤其欢迎：

- 机械设计、加工、测量、检验相关的经验与案例。
- 对规范、术语、图样表达或流程描述的修正建议。
- 对页面结构、阅读顺序、命名方式和排版细节的优化建议。
- 能让内容更严谨、更易懂、更接近实际工作的补充。
